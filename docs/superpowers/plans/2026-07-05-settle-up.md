# tally-up — Settle-Up Implementation Plan (Phase 6)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The settle-up feature from `docs/architecture.md` §5b: a greedy transfer plan computed from net positions, a `GET /settle-plan` endpoint returning it with its snapshot `seq`, optimistic-concurrency enforcement when a settlement records against a plan (`plan_seq`), and the client settle screen.

**Architecture:** The plan is a **read-model artifact** — the ledger only records settlements that actually happened. `SettlePlan` is a pure function in `internal/ledger` (greedy: repeatedly match largest debtor with largest creditor; ≤ n−1 transfers; truly minimal is NP-hard, greedy is the documented choice). A settlement recorded with `plan_seq` set is accepted only if the group's ledger hasn't moved since that snapshot; the check is made sound under concurrency by `SELECT … FOR UPDATE` on the group row, which serializes plan-checked settlements per group. Settlements without `plan_seq` (manual/partial) skip the check and remain always-valid, per the architecture.

**Tech Stack:** unchanged (Go, pgx, rapid; Next.js client).

**Prerequisites:** Phase 1–2 plan, Reads + Reversals plan, and Client plan are executed. Key interfaces consumed: `ledger.Posting`, `store.BalanceSnapshot{Balances, AsOfSeq}`, `store.GetBalances`, `store.EntryInput`, `store.CreateEntry`, `api` handler helpers, web `lib/api.ts` + `useGroupData`.

## Global Constraints

- Money is integer yen in `int64` / integer `number`. No floats.
- `SettlePlan` is deterministic: same balances → byte-identical transfer list (ties broken by ascending member UUID bytes).
- Executing a proposed plan in full drives every balance to exactly zero (architecture invariant 9) — property-tested.
- The transfer count is at most (number of nonzero balances − 1).
- A `plan_seq`-checked settlement is atomic-or-rejected: `409` with the current `as_of_seq` when stale; the client recomputes and re-proposes, never force-applies a stale plan.
- `plan_seq` is stored on the settlement entry (column exists since Phase 1–2).
- Every client-minted ID is UUIDv7 via `uuidv7()` from `web/lib/uuidv7.ts` (added in the Next.js client plan's Task 2) — never `crypto.randomUUID()` (v4-only).
- Branch: `feat/issue-7-settle-up`.

```go
package ledger

import (
	"bytes"
	"fmt"
	"sort"

	"github.com/google/uuid"
)

// Transfer is one proposed payment in a settle-up plan.
type Transfer struct {
	From   uuid.UUID `json:"from"`
	To     uuid.UUID `json:"to"`
	Amount int64     `json:"amount"`
}

// SettlePlan proposes transfers that zero all net positions. Greedy: always
// match the largest debtor with the largest creditor — at most n−1 transfers.
// (Truly minimal transfer count is NP-hard; greedy is the documented choice,
// architecture.md §5b.) Deterministic: ties break by ascending member UUID.
func SettlePlan(balances []Posting) ([]Transfer, error) {
	var sum int64
	for _, b := range balances {
		sum += b.Amount
	}
	if sum != 0 {
		return nil, fmt.Errorf("balances sum to %d, not 0 — ledger integrity problem", sum)
	}

	var creditors, debtors []Posting
	for _, b := range balances {
		switch {
		case b.Amount > 0:
			creditors = append(creditors, b)
		case b.Amount < 0:
			debtors = append(debtors, b)
		}
	}
	// Largest first; ties by member id so the plan is reproducible.
	sort.Slice(creditors, func(i, j int) bool {
		if creditors[i].Amount != creditors[j].Amount {
			return creditors[i].Amount > creditors[j].Amount
		}
		return bytes.Compare(creditors[i].MemberID[:], creditors[j].MemberID[:]) < 0
	})
	sort.Slice(debtors, func(i, j int) bool {
		if debtors[i].Amount != debtors[j].Amount {
			return debtors[i].Amount < debtors[j].Amount // most negative first
		}
		return bytes.Compare(debtors[i].MemberID[:], debtors[j].MemberID[:]) < 0
	})

	transfers := []Transfer{}
	ci, di := 0, 0
	for ci < len(creditors) && di < len(debtors) {
		credit, debt := creditors[ci].Amount, -debtors[di].Amount
		amount := min(credit, debt)
		transfers = append(transfers, Transfer{
			From: debtors[di].MemberID, To: creditors[ci].MemberID, Amount: amount,
		})
		creditors[ci].Amount -= amount
		debtors[di].Amount += amount
		if creditors[ci].Amount == 0 {
			ci++
		}
		if debtors[di].Amount == 0 {
			di++
		}
	}
	return transfers, nil
}
```

```go
package api

import (
	"encoding/json"
	"net/http"

	"github.com/google/uuid"

	"tallyup/internal/ledger"
)

// The settle plan is a read-model artifact (architecture §5b): computed from
// a balance snapshot, never stored. as_of_seq is the optimistic-concurrency
// token a recording settlement passes back as plan_seq.
func (s *Server) handleSettlePlan(w http.ResponseWriter, r *http.Request) {
	groupID, err := uuid.Parse(r.PathValue("group_id"))
	if err != nil {
		httpError(w, http.StatusBadRequest, "invalid group id")
		return
	}
	snap, err := s.store.GetBalances(r.Context(), groupID)
	if err != nil {
		httpError(w, http.StatusInternalServerError, "balance read failed")
		return
	}
	balances := make([]ledger.Posting, len(snap.Balances))
	for i, b := range snap.Balances {
		balances[i] = ledger.Posting{MemberID: b.MemberID, Amount: b.Balance}
	}
	transfers, err := ledger.SettlePlan(balances)
	if err != nil {
		// Non-zero-sum balances mean ledger corruption — surface loudly.
		httpError(w, http.StatusInternalServerError, err.Error())
		return
	}
	w.Header().Set("Content-Type", "application/json")
	json.NewEncoder(w).Encode(map[string]any{
		"transfers": transfers,
		"as_of_seq": snap.AsOfSeq,
	})
}
```

```go
func TestPlanSeq_ConcurrentCheckedSettlements_OneWins(t *testing.T) {
	s := TestStore(t)
	seedReadGroup(t, s)
	addExpense(t, s, uuid.New(), rYuto, 8000, []uuid.UUID{rMemA, rMemB})

		t.Fatal(err)

	var wg sync.WaitGroup
	errs := make(chan error, workers)
	for i := 0; i < workers; i++ {
		wg.Add(1)

			defer wg.Done()

		}()
```

- [ ] **Step 2: Settle screen**

`web/app/g/[groupId]/settle/page.tsx`:

```tsx
"use client";

import Link from "next/link";
import { use, useCallback, useEffect, useState } from "react";
import { PlanStaleError, addSettlement, getGroup, getSettlePlan } from "@/lib/api";
import type { Group, SettlePlanResponse } from "@/lib/types";
import { uuidv7 } from "@/lib/uuidv7";

export default function SettlePage({ params }: { params: Promise<{ groupId: string }> }) {
  const { groupId } = use(params);
  const [group, setGroup] = useState<Group | null>(null);
  const [plan, setPlan] = useState<SettlePlanResponse | null>(null);
  const [notice, setNotice] = useState("");
  const [busy, setBusy] = useState<string | null>(null); // "from:to" of in-flight transfer

  const load = useCallback(async () => {
    setPlan(await getSettlePlan(groupId));
  }, [groupId]);

  useEffect(() => {
    getGroup(groupId).then(setGroup).catch(() => {});
    void load();
  }, [groupId, load]);

  if (!group || !plan) return <main className="p-4 text-gray-500">Loading…</main>;
  const nameOf = (id: string) => group.members.find((m) => m.id === id)?.name ?? "?";

  async function markPaid(from: string, to: string, amount: number) {
    setBusy(`${from}:${to}`);
    setNotice("");
    try {
      await addSettlement(groupId,
        { payerId: from, counterpartyId: to, amount, planSeq: plan!.as_of_seq },
        uuidv7());
      await load(); // re-propose from the new snapshot
    } catch (err) {
      if (err instanceof PlanStaleError) {
        setNotice("Balances changed — here's the updated plan.");
        await load();
      } else {
        setNotice(err instanceof Error ? err.message : "failed to record payment");
      }
    } finally {
      setBusy(null);
    }
  }

  return (
    <main className="mx-auto max-w-md p-4">
      <h1 className="mb-1 text-xl font-bold">Settle up</h1>
      <p className="mb-4 text-sm text-gray-500">{group.name}</p>

      {notice && <p className="mb-3 rounded-lg bg-amber-50 p-3 text-sm text-amber-800">{notice}</p>}

      {plan.transfers.length === 0 ? (
        <p className="rounded-xl border p-6 text-center text-lg">All settled 🎉</p>
      ) : (
        <div className="flex flex-col gap-2">
          {plan.transfers.map((tr) => (
            <div key={`${tr.from}:${tr.to}`} className="flex items-center justify-between rounded-xl border p-4">
              <span>
                <strong>{nameOf(tr.from)}</strong> pays <strong>{nameOf(tr.to)}</strong>
                <span className="ml-2 text-lg">¥{tr.amount.toLocaleString("ja-JP")}</span>
              </span>
              <button
                className="rounded-lg bg-green-600 px-4 py-2 text-sm font-semibold text-white disabled:opacity-50"
                disabled={busy !== null}
                onClick={() => markPaid(tr.from, tr.to, tr.amount)}
              >
                {busy === `${tr.from}:${tr.to}` ? "…" : "Mark paid"}
              </button>
            </div>
          ))}
        </div>
      )}

      <Link href={`/g/${groupId}`} className="mt-6 block text-center text-sm text-blue-600">
        ← back to {group.name}
      </Link>
    </main>
  );
}
```
